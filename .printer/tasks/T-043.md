Prereq for T-040 verbose tool reporting. Today agent/mod.rs deliberately runs claude with --print --output-format json (single result object) and avoids --verbose because it switches stdout to a streamed JSON array (see comment at agent/mod.rs:173-198). To surface per-tool activity we need a streaming parser. Add an --output-format stream-json (+--verbose) code path: a new parse fn over newline-delimited events (system init, assistant messages with tool_use blocks, final result with usage). Keep the existing single-object path for the default (non-verbose) mode. Expose parsed tool-use events to Session so it can print them. Unit-test the parser against a captured NDJSON sample.
