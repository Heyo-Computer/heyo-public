Investigation T-042 finding: the review prompt (prompts.rs render_review_body step 4 + ## Verification) already tells the agent to click-test via the computer skill and to say so explicitly when $WAYLAND_DISPLAY is unset — but nothing ENFORCES it: a PASS can still be emitted with UI left unverified. Add verdict gating so that when the diff has a UI/web surface and no display was available (no click-test performed), the verdict is capped at PARTIAL rather than PASS, making unverified UI non-silent. Touch points: parse_verdict / verdict handling in review.rs and the ## Verdict rubric in render_review_body. Independent of T-044/T-045.
