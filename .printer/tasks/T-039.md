## Notes

- 2026-05-29T18:03:38Z [sarocu] PLAN — new `printer spec` command to mark a spec completed/cancelled.

A "spec" is tracked via its per-spec checkpoint at .printer/exec/<key>.json (exec.rs: Phase enum Planning/Running/ReviewPending/Done, checkpoint_path_for_spec). "Mark completed/cancelled" = set/force the checkpoint phase.

Files & changes:
- cli.rs: add `Spec(SpecArgs)` top-level command with `#[command(subcommand)] SpecCommand { Complete(SpecTargetArgs), Cancel(SpecTargetArgs) }`. SpecTargetArgs { spec: PathBuf, cwd: Option<PathBuf> }. Document under cli.rs Command enum.
- exec.rs: add `Phase::Cancelled` (#[serde(rename_all=snake_case)] -> "cancelled"). Add pub fn mark_spec_phase(cwd, spec, Phase) -> Result<()>: canonicalize cwd+spec, p = checkpoint_path_for_spec(&cwd,&spec); load-or-Checkpoint::new; set phase; updated_at=now; save. For Complete -> Phase::Done; Cancel -> Phase::Cancelled. Print confirmation + path.
- exec.rs decide()/action_from_phase(): handle Phase::Cancelled -> Action::AlreadyDone (so a later `printer exec` on a cancelled spec is a no-op with a clear message; re-running fresh still goes through FreshAfterDone-style archive — extend the `cp.phase == Phase::Done` arm in decide (false,Some) to also match Cancelled so re-exec archives + restarts).
- exec.rs print_history / list_checkpoints already render phase via Debug — Cancelled shows up automatically. Add a one-line filter note if desired.
- main.rs: dispatch Command::Spec -> match Complete/Cancel -> exec::mark_spec_phase.

Verify:
- cargo build; cargo test (extend exec.rs tests: mark_spec_phase writes Cancelled; decide() treats Cancelled like Done for AlreadyDone + FreshAfterDone).
- manual: `printer exec spec.md` to seed a checkpoint, `printer spec cancel spec.md`, inspect .printer/exec/<key>.json shows "cancelled", re-run `printer exec spec.md` reports already-done/archives.
